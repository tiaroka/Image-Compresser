# 画像圧縮ツール (Image Compressor Tool)

このプロジェクトは、ブラウザだけで動作するシンプルかつ高機能な画像圧縮ツールです。サーバサイドの処理や特殊なセットアップは不要で、HTMLファイルをそのまま開くだけで利用できます。

---

## 特徴 (Features)

1. **簡単操作**  
   - ドラッグ＆ドロップまたは「写真を選択」ボタンから画像ファイルを投入するだけで、簡単に圧縮・リサイズが可能。  
   - PCでもスマートフォンでも直感的に使えます。

2. **柔軟な圧縮設定**  
   - 圧縮品質（1～100）の調整。  
   - 最大幅（px）の指定によるリサイズ。  
   - Exif情報（位置情報やその他メタデータ）の削除・保持を選択可能。  
   - 画像の回転や反転などの向きを自動で補正（保持）できます。

3. **複数ファイルの並列処理**  
   - 複数の画像ファイルを同時に処理し、進捗バーで処理状況がわかりやすい。  
   - （設定された最大数の）ワーカーが並列で処理を行うため、効率的に圧縮できます。

4. **ダウンロード & 共有機能**  
   - 圧縮後の画像は個別にダウンロード（PC）または共有（スマホ）可能。  
   - 10枚以上の画像がある場合には自動でZIPファイルを生成してまとめてダウンロードできるので便利です。

5. **軽量で依存が少ない**  
   - 基本的には1つのHTMLファイルにまとまっており、Tailwind CSS（CDN経由）、EXIF.js、JSZipを利用。  
   - サーバレスで動作するので、インストールやビルドなどの手間が不要。

---

## デモ/プレビュー

- **ローカルでの利用方法**  
  1. このリポジトリをクローンまたはZIPダウンロード  
  2. `image-compressor.html`（またはリポジトリ内の該当ファイル）をダブルクリックして任意のブラウザで開く  
  3. 画像をドラッグ＆ドロップ、または「写真を選択」ボタンからファイルを指定して、ツールをお試しください

- **GitHub Pagesでホスティングする場合**  
  1. このリポジトリを自分のGitHubアカウントにフォークまたは複製  
  2. リポジトリの設定から「Pages」を有効化し、ブランチとフォルダを選択  
  3. 公開されたURL (例: `https://<ユーザ名>.github.io/<リポジトリ名>/image-compressor.html`) をブラウザで開く  

---

## 使い方 (How to Use)

1. ブラウザでファイルを開くと、設定画面とアップロード用の領域が表示されます。
2. **圧縮設定**  
   - **品質 (1-100)**: 数値が高いほど画質優先。低いほど圧縮率が高まりますが、画質が劣化します。  
   - **最大幅 (px)**: 画像の幅を指定したピクセル数に縮小します（縦横比は維持されます）。  
   - **Exif設定**:  
     - 「位置情報を削除」にチェック → GPSなど位置情報を削除  
     - 「その他のExif情報を削除」にチェック → カメラ機種名などの全Exif情報を削除  
     - 「画像の向きを保持」にチェック → 回転情報を自動検知して正しい向きに補正  
3. **画像を選択またはドラッグ＆ドロップ**  
   - サポートされている形式: `JPEG, PNG, GIF, WebP`  
   - 複数枚の画像をまとめて指定することも可能
4. **変換開始**  
   - 「変換開始」ボタンを押すと、指定した設定で順次画像の圧縮が行われます。  
   - ファイル数が多い場合、進捗バーが徐々に伸びていき、何枚目を処理中か確認できます。  
   - エラーやサポート外のファイルがある場合は警告が表示されます。
5. **ダウンロード/共有**  
   - 圧縮が完了した画像は、個別に「ダウンロード」ボタンから保存することが可能（PCの場合）。  
   - スマホでアクセスした場合は、個別画像ごとの「共有」もしくは「保存」に切り替わります。  
   - 画像が10枚以上ある場合は、ZIPファイルを自動生成し、一括でダウンロードできるボタンが表示されます。  
6. **クリア**  
   - 「結果をクリア」ボタンを押すと、アップロード済みのファイルや生成したファイルのプレビューを全て消去します。

## 依存ライブラリ (Dependencies)

- [Tailwind CSS](https://tailwindcss.com/) (CDN)  
  シンプルなユーティリティクラスベースのCSSフレームワーク

- [EXIF.js](https://github.com/exif-js/exif-js) (CDN)  
  JPEGファイルからExif情報を読み取るためのライブラリ

- [JSZip](https://stuk.github.io/jszip/) (CDN, on-demand)  
  複数画像をまとめてダウンロードする際、ZIPファイルを生成するために使用

全てCDN経由のため、事前インストール等は不要です。  
オフライン環境などでは、これらのライブラリをローカルにダウンロードしてHTML内の参照を変更すればOKです。

## よくある質問 (FAQ)

**Q. 大きすぎる画像を読み込むとどうなりますか？**  
A. ブラウザでの処理のため、あまりに巨大な画像（数百MB～）を入れるとメモリ不足になることがあります。1枚あたり50MB以下程度を目安にご利用ください。

**Q. 画像の画質があまり落ちないようにサイズだけを変更したい場合は？**  
A. 品質(1-100)を100に近い値にし、最大幅を指定することでリサイズのみ行います。

**Q. PDFやSVGなど、他の形式は使えますか？**  
A. 現在はJPEG, PNG, GIF, WebPのみサポートです。他形式をドロップした場合は警告が出て処理がスキップされます。
